Rootkits em eBPF: como um filtro de pacotes vira uma porta dos fundos

linux ebpf

eBPF nasceu como uma forma segura de instrumentar o kernel sem precisar recompilá-lo. Ferramentas como Falco, Cilium e Datadog usam a mesma capacidade — hooks em kprobes, tracepoints e chamadas de sistema — para observabilidade e defesa. O problema é que essa capacidade é neutra: qualquer coisa que dá visibilidade profunda ao defensor dá o mesmo acesso ao atacante.

O caminho de um programa eBPF

Um programa eBPF é escrito em um dialeto restrito de C, compilado para bytecode, validado pelo verificador do kernel e então JIT-compilado para código nativo. O verificador prova terminação e segurança de memória — mas não prova intenção. Um programa tecnicamente seguro ainda pode filtrar diretórios, interceptar tráfego ou negar eventos de auditoria silenciosamente.

observação de laboratório Todos os testes abaixo foram executados em uma VM isolada, sem acesso à rede externa. Não rode isso fora de um ambiente controlado.

Hook em getdents64

Um dos usos mais diretos é anexar um kprobe à syscall __x64_sys_getdents64, responsável por listar entradas de diretório. Ao filtrar o buffer retornado antes que ele chegue ao userland, é possível esconder arquivos com um padrão de nome específico — sem tocar em nenhuma estrutura do VFS. 

SEC("kprobe/__x64_sys_getdents64")
int hide_entry(struct pt_regs *ctx) {
    // captura o buffer de retorno via kretprobe
    // filtra entradas cujo d_name bate com o padrão alvo
    return 0;
}

Detecção

A defesa prática começa em bpftool prog list, que enumera todos os programas eBPF carregados junto com tipo, ID e nome. Os indicadores mais úteis:

  • Programas sem nome associado
  • Programas carregados a partir de inodes anônimos
  • Tipos incomuns (LSM, SK_SKB) sem uma aplicação legítima correspondente rodando

Nenhum desses sinais é definitivo isoladamente — mas a combinação de um programa anônimo do tipo LSM sem processo pai reconhecível é, na prática, quase sempre motivo de investigação.

Conclusão

eBPF é dual-use por natureza. A mesma API que dá ao Falco visibilidade em tempo real dá a um rootkit a capacidade de mentir para o restante do sistema sobre o que está acontecendo. Entender essa simetria é o primeiro passo pra saber o que procurar.
